# 待办清单 (Todo List) 📝

一个基于 Next.js + Supabase 的待办事项应用。

## 功能

- ✅ 添加任务
- ☑️ 标记完成
- 🗑️ 删除任务
- 💾 数据持久化（刷新不丢失）

## 技术栈

- **框架**: Next.js 14 (App Router)
- **数据库**: Supabase
- **部署**: Vercel

## 本地开发

### 1. 创建 Supabase 项目

1. 去 [supabase.com](https://supabase.com) 注册并创建项目
2. 在 SQL Editor 中运行以下语句创建表：

```sql
CREATE TABLE todos (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  text TEXT NOT NULL,
  completed BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIME NOW()
);
```

3. 在 Project Settings → API 中获取：
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`

### 2. 配置环境变量

在项目根目录创建 `.env.local` 文件：

```env
NEXT_PUBLIC_SUPABASE_URL=你的Supabase项目URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的Anon Key
```

### 3. 安装并运行

```bash
npm install
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000)

## 部署到 Vercel

1. 把代码推送到 GitHub
2. 在 Vercel 导入项目
3. 在 Vercel 的 Environment Variables 中添加：
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. Deploy!

## 环境变量说明

| 变量名 | 说明 |
|--------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 项目 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase 公开的 Anon Key（安全，可在前端使用） |
